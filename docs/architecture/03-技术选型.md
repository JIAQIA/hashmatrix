# 03 · 技术选型

原则：**标准开源优先**；信创只作用于「对接外部数据源」与「最终部署环境」，不左右平台自身开源选型。

## 1. 选型菜单（首选 + 备选）

| 能力域 | 首选（推荐） | 备选 | 信创备注 |
|---|---|---|---|
| 流采集 | Flink + Kafka + Flink-CDC | SeaTunnel / Debezium | 可信创编译 |
| 批采集 | SeaTunnel | DataX | 连接器插件化，适配达梦/金仓 |
| 湖格式 | Apache Paimon | Iceberg / Hudi | Paimon 流式更新强，契合 Flink |
| OLAP/实时数仓 | Apache Doris | StarRocks / ClickHouse | Doris 信创案例多 |
| 对象存储 | MinIO | Ceph | 替代云对象存储 |

> **元数据 / 元模型引擎说明**：本平台数据治理的核心是**运行期可由用户自定义的元模型引擎**（元类带继承、属性约束、关系基数、分类树、草稿/发布、版本、平台公共 + 租户私有作用域）。OpenMetadata / DataHub 的类型模型是**代码态、不支持 UI 自定义带继承的类型**，不契合该诉求；故 governance 走**自研元模型引擎**——以 **Apache Atlas 的 TypeDef 体系为设计蓝本**（entityDef+superTypes 继承 / classificationDef / relationshipDef 带基数），落 Spring Boot + PostgreSQL(JSONB) + Elasticsearch(检索)。OpenMetadata / Atlas / DataHub **降级为元数据采集连接器与血缘解析的可选来源**，与 Connector SPI 并存桥接。若不自研，唯一够格直接用的开源是 **Apache Atlas**（须接受其重型存储 JanusGraph/HBase 与多租户/信创改造成本）。自研利于**元模型多租户作用域、信创适配、甲方可接手自维护**。详见 governance 仓选型 Issue。

## 2. 与硬指标对标

| 硬指标 | 支撑组件 | 设计要点 |
|---|---|---|
| 实时入湖 ≤5s（70 万条/s，单条 1KB） | Flink + Kafka + Paimon | 流式入湖、分区与并行度调优、背压监控 |
| 批量触发→入湖 ≤15min | SeaTunnel / Spark | 并行抽取、增量切片 |
| 并发写入连接 ≥100 | Kafka + Flink | 多分区、连接池 |
| 存储 ≥6PB 主备（各 ≥3PB） | MinIO + Doris | 主备双集群、纠删码、动态扩展 |
| IO ≥70 万条/s（≤500MB/s） | Doris / Paimon | 列存压缩、批量导入 |
| 实时数仓并发 ≥1000 | Doris | 资源组隔离、查询排队 |
| 高并发点查 1 万 QPS，TP95 ≤100ms | Doris（主键模型/前缀索引）+ Redis | 点查模型 + 缓存旁路 |
| 向量 100T、3 副本、8% 损耗→主备 8 节点 | Milvus | 分片副本、磁盘索引 |
| 隐私计算：千万级求交 ≤100s；亿级匿踪亚秒级 | SecretFlow | PSI/OT 算法、节点互联编排 |
| 数据模型：小时≤30min/日≤2h/月≤4h | DolphinScheduler + Spark/Flink | 依赖编排、资源调度、SLA 告警 |
| 分类分级标签 ≥100 条、50 并发打标 | security 服务 + 规则引擎 | 规则可热更新（Nacos）、并发任务池 |
| 可用性 99.99% | K8s 多副本 + 主备 | 无单点、HPA、探针、滚动升级 |

## 3. 多语言现实

主栈 **Java + TS**，但承认多语言：

- **Java**：网关下游各业务服务、采集/计算控制面。
- **TS**：WebUI、可视化编排前端。
- **Python**：隐私计算（SecretFlow）、AI/向量、部分数据科学 —— 作为**独立 submodule（privacy）隔离**，通过 API/gRPC 与 Java 服务交互，不混入 JVM 工程。
- **SQL/Scala**：Flink/Spark 作业。
