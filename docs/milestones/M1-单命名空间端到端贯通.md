# M1 · 单命名空间端到端贯通（统一里程碑）

> 范围：跨 11 个仓库的**第一个开发里程碑**协调总纲。各仓以**同名 Milestone**承接本文对应小节。
> 定位：可向上**阶段性演示汇报**的「骨架贯通」——不保证完整可用，但**基础骨架不可返工、架构不可错误**。
> 红线：本文与各仓 Issue/分支/提交均为公开内容，禁止任何甲方可识别信息与项目机密（见各仓 `CLAUDE.md`）。

---

## 1. 目标与验收

**统一目标**：在本地 kind 单命名空间内，把已联调成功的各子系统以**一个租户上下文**端到端跑通，侧栏一级/二级导航**有功能可见**、各模块**可演示**。

**演示验收（Demo Acceptance）**：

1. `tools/local-infra` 起 kind → 一条 `helm install` 把平台装入**单个 namespace**（如 `demo`），**纳入 demo 链路的各服务（含 Keycloak/PG 等 dev 级依赖）均真实 chart + 镜像 + 部署、Pod `Running` 且 readiness 真实绿**（非仅 NOTES.txt 占位）。
2. 经 gateway(APISIX) 走 **Keycloak 标准 OIDC 登录** → 注入 `X-Tenant-*` → 后端 `starter-tenant` 解析出**一个真实租户上下文**。
3. control-plane **开通一个租户**（真实建 Keycloak org + 租户管理员；compute/data/secrets 可 stub）。
4. webui console 加载，**emerald 品牌**生效，**九大模块侧栏 L1/L2 可见**，每模块**入口页可点开演示**，「组织管理」区段可见（按角色门控）。
5. governance `/api/meta/search` 返回脱敏 mock；data-foundation / security / privacy 各有**一个 demoable 端点/probe 绿**。
6. **端到端纵切贯通链（M1 的 definition of done，必须有人认领并跑通）**：浏览器 → gateway（Keycloak OIDC 登录 + 注入 `X-Tenant-*`）→ governance `/api/meta/search` →(PG/ES)→ **webui 数据目录页渲染该真实返回**；并由 control-plane **真实开通一个租户**。其余各服务**经网关可达、readiness 真实绿**（业务暂 probe/mock 级可接受）。

**非目标（明确后置，避免镀金）**：完整业务功能；完整 SC-6 行/列裁决（M1 仅行级兜底 + 库表级角色门）；OIDC 企业 IdP 联邦；真实 compute/data/secrets Provisioner；多租户并发切换的完整实现；CDC/Pact 契约测试；**On YARN 计算调度后端**（M1 默认实装 On K8s，On YARN 为存量 Hadoop 客户私有化的**后置交付 profile**，仅预留抽象——见 AD-18 与下方 data-foundation/control-plane 不返工不变量）。

## 2. 已确认的关键决策（不可返工）

| # | 决策 | 结论 | 影响面 |
|---|---|---|---|
| **D1** | 用户/租户/Account 模型 | **单 Keycloak User（唯一登录）+ 多 Org Membership + 每租户角色**。「Account per tenant」= 该 User 在该租户的 Membership+角色，**不另建凭证实体**。 | 身份全链路；M1 数据模型/契约**即按"可属多租户"预留**，实现先返回单租户 |
| **D2** | 租户切换机制 | 切换 = **重新换 token**（Keycloak 指定 organization），`X-Tenant-Id` **始终唯一**，下游 30+ 服务**零改动** | 复杂度收敛在登录/网关边缘 |
| **D3** | 白标隔离粒度 | **部署级注入**（`config.js`），同部署内风格统一，**绝不按租户运行期换肤**（代码现状已如此） | webui；维持现 `packages/brand` 机制 |
| **D4** | 默认品牌 | **Emerald 绿** `colorPrimary #059669 / colorSecondary #10b981`；appName=**HashMatrix**、通用占位 logo（合规） | webui `brand.default.json` + 部署期默认值 |
| **D5** | Helm 职责 | **子仓交付 Dockerfile/image，主仓 `deploy/` owns 全部 charts**（消解 README 的自相矛盾） | 所有子仓 + 主仓 deploy |
| **D6** | 身份后端深度 | M1 只做**真实 Keycloak OIDC 登录 + control-plane 真实 IdentityProvisioner**；compute/data/secrets 留 stub；OIDC 联邦、完整 SC-6 后置 | control-plane / gateway / libs-java |

> **契约必须同步修订**：`contracts/icd/tenant-context-headers-icd.md §3.4` 现写死「多 org 成员→403」，与 D1/D2 冲突 → 改为「M1 暂解析到单活动租户、但模型/头预留多 membership；切换经重新换 token」。否则代码与契约打架。

## 3. 统一端口分配基线（解决"同机并行开发端口冲突"）

> 现状问题：6 个服务全默认 8080、4 个 PG 全占 5432、Keycloak 8080/8081 打架、MinIO 与 node-mock 抢 9000。
> 约定：**应用 HTTP = 80xx，管理/actuator = 90xx**（`management.server.port` 独立）；**共享中间件单实例**，各服务用**独立 database** 而非各起一个容器。端口一律 `${ENV:默认}` 可覆盖，由根 `Makefile` 驱动，**不再手维护**。

### 应用服务

| 服务 | App HTTP | 管理端口 | 备注 |
|---|---|---|---|
| gateway (APISIX) | **9080**(proxy) / 9180(admin, standalone 关) | — | 前门，沿用 APISIX 习惯端口 |
| control-plane | **8081** | 9081 | 当前无 `server.port`（默认 8080）→ 显式设 |
| governance | **8082** | 9082 | 同上 |
| security | **8083** | 9083 | 同上 |
| data-foundation (app) | **8084** | 9084 | `application.yml` 硬编码 8080 → 改 `${SERVER_PORT:8084}` |
| privacy orchestrator-java | **8086** | 9086 | 硬编码 8080 → 改 `${SERVER_PORT:8086}` |
| privacy engine-py (FastAPI) | **8087** | —(`/healthz`) | 覆盖 `ENGINE_PORT`，移出 8000 |
| privacy node-mock | **8088** | —(`/healthz`) | 覆盖 `NODE_PORT`，让出 9000 给 MinIO |
| platform-common（建成后） | **8089** | 9089 | 预留槽 |

## 3.5 M1 贯通主线（跨仓集成 · 主仓作为 owner）⭐

> 各仓 brief 只保证"本服务 boot+probe+镜像"；**把它们接起来的跨仓胶水必须单独认领**，否则 M1 会沦为"11 个东西各自起来 + 界面看得到但没接通"。本主线由**主仓**统筹，对应主仓 tracking issue「**M1 贯通主线 · 端到端集成**」。现状实测缺口：gateway 路由全指向 `mock-upstream`、Keycloak/PG 在 umbrella 中 `enabled:false` 且无 in-cluster 部署、webui 不在 umbrella、SDK 仍 `mockSessionStore`。

| WP | 内容 | 触及仓 |
|--|--|--|
| **I1 In-cluster 基础设施** | Keycloak + PG（dev 级、非高可用）作为 umbrella 可选依赖在 demo ns 跑起来；realm/client 预置 | 主仓 deploy（+ control-plane/gateway 消费） |
| **I2 gateway 路由真实后端** | apisix routes/upstreams 指向 in-cluster Service（governance/control-plane…），弃 `mock-upstream`；OIDC `discovery` 指向集群内 Keycloak | gateway |
| **I3 webui 进集群 + 指向网关** | 建 `deploy/charts/webui`（nginx）；`config.js` 注入 in-cluster gateway 的 `api.baseUrl` + `oidc.authority`；SDK 用真实 OIDC session 替 `mockSessionStore` | 主仓 deploy + webui |
| **I4 纵切贯通链 + 集成验收** | 跑通验收 §1.6 那条链并落一个集成自检（浏览器→gateway(OIDC+租户)→governance→PG→webui 渲染 + control-plane 开通一租户） | 主仓 owner，串 gateway/control-plane/governance/webui |
| **I5 各服务真实化收口** | 纳入 demo 链路的服务都补真实子 chart + 镜像 + 接线（非占位），经网关可达、readiness 真实绿 | 主仓 deploy + 各后端仓 |

**各仓接点**（在其 brief 之外、由本主线收紧的依赖/验收）：gateway=I2；webui=I3；control-plane=依赖 I1 的 Keycloak、并被 I4 串联；governance=I4 纵切链的后端（真实经网关被调）；主仓 deploy=I1/I3/I5。

## 6. 风险与待澄清（迭代中持续澄清）

- **R1 身份模型**：D1/D2 已定，但 `tenant-context-headers-icd.md` 必须同步改，否则"多 org→403"会拦住未来多部门租户。**M1 内先落档 + 改契约**。

- **R3 Helm 占位**：7 个子 chart 仅 NOTES.txt，单 ns demo 需先有黄金模板可复制；privacy 子 chart 是唯一已真实参考。
- **R4 platform-common 空壳**：无 pom，image 产不出，会拖累 umbrella；M1 必须补最小骨架。
- **待澄清（不阻塞 M1 骨架）**：OIDC 企业 IdP 联邦的具体接入时机；完整 SC-6 行/列裁决的优先级；多租户切换的前端交互细节。这些在 M1 推进中与产品持续澄清，**数据模型已按可扩展预留**。
